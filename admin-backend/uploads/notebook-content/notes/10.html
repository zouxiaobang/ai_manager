🛠️ 2.0+ 最新标准 Bean 配置写法

你完全可以在 OpenAiImageOptions 构建时直接把阿里灵积的 baseUrl 和 apiKey 注入进去，它在底层就会使用新版的官方客户端去发起正确的请求，代码如下：

Java

import org.springframework.ai.openai.OpenAiImageModel;
import org.springframework.ai.openai.OpenAiImageOptions;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class ImageModelConfig {

    @Bean
    public OpenAiImageModel openAiImageModel() {
        // 💡 2.0+ 新版：baseUrl 和 apiKey 直接内聚在了 OpenAiImageOptions 的 Builder 中
        OpenAiImageOptions options = OpenAiImageOptions.builder()
        .baseUrl("https://dashscope.aliyuncs.com/compatible-mode/v1")
        .apiKey("sk-ws-H.RERRIRD.p4Rv.MEYCIQD-VHzRtD-bWI6CEe-hF2NMwyCNbILnnwfApIx1tH6nCQIhAPR89Z0oBzE37SXDbJuZX545bvNgonIC-lEfs24ugJBq")
        // 可以在这里设置 Bean 的全局默认图片模型（万象）
        .model("wanx2.1-t2i-plus")
        .width(1024)
        .height(1024)
        .build();
        

        // 💡 直接通过 Options 实例创建 ImageModel
        return new OpenAiImageModel(options);
    }
}

📝 为什么这样做现在能成功？

  1. 统一了配置载体：新架构下不再有第三方网络层的二次包装，OpenAiImageOptions 会把你的配置直接转换成 OpenAI 官方 SDK 的 ImageGenerateParams。
  2. 消除了路径硬拼接：官方新版 SDK 在拿到你显式传入的 https://dashscope.aliyuncs.com/compatible-mode/v1 后，能够更标准地去跟后置的生成动作做路由契合，从而解决了老版本自动装配时的 404 Unknown 异常。

用这个全新的 @Bean 方法替换后，直接去 Controller 里注入并调用，就能顺利出图了。
